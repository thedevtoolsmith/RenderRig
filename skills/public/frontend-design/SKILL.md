# Frontend Design

Build real, working frontend code that is visually intentional and production-ready.

## Workflow

1. Clarify purpose, audience, and constraints from the user request.
2. Choose one bold aesthetic direction before writing code.
3. Define the memorable signature element that differentiates the design.
4. Implement full working UI (HTML/CSS/JS or framework-specific code) with cohesive visual language.
5. Verify responsiveness, accessibility, and interaction quality on desktop and mobile.

## Art Direction

Commit to a clear point of view. Favor intentional extremes over generic middle-ground output.

- Pick a design tone such as editorial, brutalist, retro-futuristic, industrial, playful, luxury, or organic.
- Maintain coherence: type, color, spacing, motion, and decoration should support the same concept.
- Ensure the design has one unforgettable element (layout gesture, hero treatment, interaction pattern, or typography system).

## Implementation Standards

- Build production-grade, functional code rather than mockups.
- Use expressive typography pairings; avoid generic defaults (`Arial`, `Roboto`, `Inter`, system stack only).
- Define a focused palette with CSS variables; avoid timid, interchangeable color themes.
- Use purposeful animation for key moments (load sequence, staggered reveal, scroll entry, high-value hover states).
- Compose layouts with intention: asymmetry, overlap, negative space, or controlled density as needed.
- Add atmospheric background detail (mesh gradients, textures, geometric layers, grain, custom shadows, patterned fields) when aligned with the concept.
- Match code complexity to the visual direction: maximalist concepts need richer implementation; refined minimalist concepts need precision and restraint.

## Anti-Patterns

- Do not generate cookie-cutter component layouts.
- Do not default to overused AI aesthetics (including purple-gradient-on-white tropes).
- Do not mix unrelated visual styles that break conceptual cohesion.
- Do not ship designs that look unfinished, unresponsive, or inaccessible.

## Delivery Checklist

- Confirm the UI works end-to-end.
- Confirm desktop and mobile behavior is solid.
- Confirm visual hierarchy and readability are clear.
- Confirm states are designed (hover, focus, active, disabled, loading where relevant).
- Confirm the final output clearly reflects the chosen design concept.
